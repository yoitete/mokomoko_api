# MokoMoko 枕サービスへの転換 実装計画書

## 1. プロジェクト概要

### 1.1 背景

現在の「MokoMoko」は毛布特化の投稿共有サービスとして運用中。  
毛布は「触り心地」が主な評価軸であるのに対し、枕は「高さ・硬さ・素材・寝姿勢・体への影響」など多次元的な評価が可能であり、サービスとしての機能拡張性と実用性が大きく向上する。

### 1.2 コンセプトの変更

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| テーマ | 毛布 | 枕 |
| キャッチコピー | もこもこで見つける小さな幸せ | もこもこで見つける最高の眠り（案） |
| 核心的価値 | 触り心地の共有 | 自分に合った枕を見つける意思決定支援 |
| ターゲット | 毛布好きな人 | 枕選びに悩むすべての人 |

### 1.3 サービス名について

「MokoMoko」は枕でも十分に通用する名前（もこもこした枕のイメージ）。  
変更不要だが、サブタイトルやコピーは見直す。

## 2. 既存機能の改修

### 2.1 データモデルの変更

#### postsテーブルの変更

```
【カラム変更】
- description → そのまま活用（枕の使用感を記述）

- price → そのまま活用

【カラム追加】

- purchase_url: string（購入リンク ※任意）
```

```

    int id PK
```

## 3. 新規機能の追加

### 3.1 レーダーチャート（触感・性能の可視化）

**概要:** 投稿ごとに6軸のスコアをレーダーチャートで表示する。

**6つの評価軸:**
1. やわらかさ（Softness）
2. サポート力（Support）
3. 通気性（Breathability）
4. 耐久性（Durability）
5. 首フィット感（Neck Fit）
6. 洗いやすさ（Washability）

**実装方針:**
- フロントエンド: Rechartsの`RadarChart`コンポーネントを使用
- バックエンド: `pillow_scores`テーブルから取得、投稿APIにネストして返却
- 投稿フォーム: 各軸1〜5のスライダーUI

**技術的アピールポイント:**
- データビジュアライゼーションライブラリの活用
- ネストされたリソースのAPI設計（posts → pillow_scores）

---

### 3.2 枕マッチング診断

**概要:** 5問の診断で「あなたにぴったりの枕」を既存投稿からおすすめ。

**診断フロー:**

```
Q1. あなたの主な寝姿勢は？
    → 仰向け / 横向き / うつ伏せ / 寝返りが多い

Q2. 体格は？
    → 小柄 / 普通 / 大柄

Q3. 肩こり・首の悩みはある？
    → 肩こりがひどい / ストレートネック気味 / 特になし

Q4. 好みの硬さは？
    → ふわふわ柔らかめ / しっかり硬め / どちらでもない

Q5. 予算は？
    → 〜3,000円 / 〜10,000円 / 10,000円以上 / 気にしない
```

**マッチングロジック（スコアリング方式）:**

```
各投稿に対してマッチスコアを算出:

score = 0

// 寝姿勢の一致
if post.sleeping_style == user_answer.sleeping_style → score += 30

// 体格 × 高さの相性
小柄 → 低め(5-8cm)推奨、大柄 → 高め(10-15cm)推奨 → score += 20

// 悩み × スコアの相性
肩こり → support_score, neck_fit_score が高いものを優先 → score += 20

// 硬さの好み × softness_score
柔らかめ好き → softness_score 4-5 を優先 → score += 15

// 予算
予算内 → score += 15

→ score上位5件を「おすすめ枕」として表示
```

**実装方針:**
- フロントエンド: ステップ形式の診断UI（React state管理）
- バックエンド: `GET /api/v1/matching?sleeping_style=...&body_type=...` でスコアリングしてソート返却
- 診断結果は`user_profiles`に保存し、次回訪問時にパーソナライズ表示にも活用

**技術的アピールポイント:**
- アルゴリズム設計と説明ができる（面接で頻出の話題）
- クエリパラメータベースのフィルタリングAPI
- ステップUIのstate管理

### 3.3 枕比較機能

**概要:** 2つの枕を横並びで比較できるUI。

**比較項目:**
- 画像
- 価格
- 素材
- 高さ
- 寝姿勢
- レーダーチャート（重ね合わせ表示）
- お気に入り数
- ブランド

**実装方針:**
- フロントエンド: 投稿一覧・詳細に「比較に追加」ボタンを配置、最大2件まで選択 → 比較ページへ遷移
- 比較対象はURLクエリパラメータで管理（`/compare?ids=1,2`）→ シェア可能
- レーダーチャートは2つのデータを重ねて表示（Rechartsで対応可能）

**技術的アピールポイント:**
- 複数リソースの同時取得とレイアウト設計
- レスポンシブ対応（モバイルでは縦並び切り替え）

### 3.4 コメント機能

**概要:** 投稿に対するコメント投稿・表示機能。

**想定されるコメント例:**
- 「この枕、洗濯機で洗えますか？」
- 「半年使った感想ですが、へたりは少ないです」
- 「横向きで寝ると高さがちょうどいいですね」

**実装方針:**
- バックエンド: `CommentsController`（CRUD）、`posts`にネストしたルーティング
- フロントエンド: 詳細ページ下部にコメント一覧 + 投稿フォーム
- ページネーション: 最新10件表示 + 「もっと見る」

**技術的アピールポイント:**
- ネストされたリソースのRESTful API設計
- N+1問題の回避（includes / eager_load）
- 楽観的UI更新（SWRのmutate）

### 3.5 SNSシェア + OGP動的生成

**概要:** 投稿をX（旧Twitter）やLINEでシェア可能にし、投稿ごとに動的OGP画像を生成。

**OGP画像に含める情報:**
- 枕の写真
- タイトル
- レーダーチャートのサマリー（例: 「やわらかさ ★★★★☆」）
- MokoMokoロゴ

**実装方針:**
- Next.js App Routerの`generateMetadata`で投稿ごとにOGP meta tagを動的生成
- OGP画像: `@vercel/og`（satori）またはAPI Routeでの画像生成
- シェアボタン: X、LINE、URLコピー

**技術的アピールポイント:**
- Next.js App Routerの深い理解
- サーバーサイドでの動的画像生成

---

### 3.6 画像の複数枚対応 + ギャラリービュー

**概要:** 1投稿につき最大5枚の画像をアップロード可能にする。

**推奨する写真の種類:**
1. 枕の全体像
2. 素材のアップ（生地の質感）
3. 高さがわかる横からの写真
4. 実際に使用しているシーン
5. パッケージ・タグ（素材表記）

**実装方針:**
- バックエンド: Active Storageの`has_many_attached :images`に変更
- フロントエンド: 複数画像アップロードUI + スワイプ対応カルーセル
- サムネイル自動生成（Active Storageのvariant）

---

### 3.7 「使用○ヶ月後」経過レビュー機能

**概要:** 自分の投稿に対して、時間経過後の追加レビューを投稿できる機能。  
枕は「最初は良かったけど3ヶ月でへたった」のような経時変化が重要な商品。

**データモデル:**

```
follow_up_reviews {
    int id PK
    int post_id FK
    int user_id FK
    int months_used          （使用月数）
    text body               （レビュー本文）
    int durability_update    （耐久性の再評価: 1-5）
    datetime created_at
    datetime updated_at
}

posts ||--o{ follow_up_reviews : "has"
users ||--o{ follow_up_reviews : "writes"
```

**表示方法:**
- 投稿詳細ページにタイムライン形式で表示
- 「1ヶ月後 → 3ヶ月後 → 6ヶ月後」のように時系列で並ぶ

**技術的アピールポイント:**
- 他の枕レビューサイトにはない独自機能
- 時系列データの設計と表示
- 面接で「なぜこの機能を作ったか」のストーリーが非常に語りやすい

### 3.8 リアルタイムお気に入り更新（ActionCable）

**概要:** お気に入りボタンの数字やランキングがリアルタイムで反映される。

**実装方針:**

- フロントエンド: WebSocket接続 + SWRのmutateで即時反映

## 8. 面接でのアピール戦略

### 語れるストーリー

```
「枕は合わないと肩こりや睡眠の質に直結する身近な課題。
 でもECサイトのレビューだけでは触感や自分の体型との相性がわからない。

 → 6軸のレーダーチャートで枕の特性を可視化し、
   マッチング診断で自分に合った枕を提案する仕組みを設計した。

 → 経過レビュー機能を追加したのは、枕は使い始めと数ヶ月後で
   評価が変わる商品だから。この視点は既存サービスにはない。」
```

### 技術面で語れるポイント

- **設計判断**: なぜスコアリング方式のマッチングにしたか（MLを使わない判断の根拠）
- **DB設計**: 正規化の判断（pillow_scoresを別テーブルにした理由）
- **パフォーマンス**: N+1対策、レーダーチャートデータの取得最適化
- **リアルタイム**: ActionCableの導入と、なぜ全機能ではなくお気に入りだけに絞ったか
- **Next.js**: OGP動的生成、App Routerのmetadata API活用
- **テスト**: 新機能に対するRSpec/Jestテストの追加
